## 一、 前言
在計算機科學領域中，排序演算法是資料處理與演算法分析的基礎。無論是處理巨量數據分析還是執行複雜的數值運算，演算法的效率直接影響系統的執行效能與即時反應。本報告旨在透過 JavaScript 實作五種經典排序演算法，並開發具備動態調速功能的視覺化界面，藉此直觀地觀察並比較各演算法在不同時間複雜度下的運行邏輯與表現差異。

## 二、 排序演算法介紹與動態展示

* **氣泡排序 (Bubble Sort)** 重複走訪待排序序列，比較相鄰的兩個元素，若順序錯誤則進行交換。較大的數值會如同氣泡般逐一浮動至序列末端。  
  👉 [點此進入氣泡排序演示網頁](./0309.html)

* **選擇排序 (Selection Sort)** 在未排序的數組中尋找最小值，將其放置於已排序序列的末尾。特點是交換次數較少，但比較次數固定，不受初始資料順序影響。  
  👉 [點此進入選擇排序演示網頁](./selection.html)

* **插入排序 (Insertion Sort)** 透過建構有序序列，將未排序數據逐一插入已排序序列中的正確位置。在數據量小或接近已排序狀態時效率極高。  
  👉 [點此進入插入排序演示網頁](./insertion.html)

* **合併排序 (Merge Sort)** 採用分而治之（Divide and Conquer）策略，將數組遞迴地拆分為最小單位後再進行合併。在各種情況下都能維持穩定的運算效率。  
  👉 [點此進入合併排序演示網頁](./merge.html)

* **沈澱排序 (Sedimentation Sort)** 此演算法為基於計數排序概念的變體。透過將數值映射至相對應的空間層級（地層），模擬物理沈降過程，避免了傳統演算法中的大量比較動作。  
  👉 [點此進入沈澱排序演示網頁](./sedimentation.html)

## 三、 複雜度分析

| 排序演算法 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- |
| 氣泡排序 | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n^2)$ | $O(1)$ 至 $O(n^2)$ | $O(1)$ | 穩定 |
| 合併排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |
| 沈澱排序 | $O(n+k)$ | $O(n+k)$ | $O(k)$ | 穩定 |

## 四、 實驗設計與結果比較

### 1. 實驗設計
本實驗透過視覺化模擬界面，設定三種不同規模的隨機數組（$n=10, 100, 1000$），觀察各演算法在處理不同數據量時的執行行為。透過調整「速度滑桿」，可以量化觀察各演算法在完成排序過程中所須的步驟密度。

### 2. 結果比較
根據模擬觀測與理論推導，結果如下：
* **執行效率**：當資料量 $n$ 增加時，合併排序與沈澱排序的執行速度明顯優於其餘三種演算法。
* **交換頻率**：氣泡排序在過程中產生最多的交換動作；選擇排序雖比較次數多，但交換次數維持在 $O(n)$，表現相對穩定。
* **環境限制**：沈澱排序雖然在時間上極快，但其依賴於數值範圍（$k$）的空間分配，若數值範圍過大則會消耗較多記憶體空間。

## 五、 心得與結論
透過本次實作與數據模擬，可以觀察到不同複雜度等級演算法在效能上的顯著差異。在數據規模較小時，氣泡與選擇排序雖易於實作且邏輯直觀，但當數據量提升時，其運算時間會呈二次方增長，效率顯著下降。相較之下，合併排序展現了 $O(n \log n)$ 在處理大規模運算時的高度穩定性。

在開發視覺化界面的過程中，透過調整速度控制，能以動態方式觀察演算法內部的運行過程，如合併排序如何進行遞迴拆分。此外，自定義的沈澱排序法也展現了利用空間映射優化效率的可能性。總結而言，選擇演算法時應根據資料規模、記憶體限制以及資料特性，在時間與空間複雜度之間做出合理的權衡。
