## 一、 前言
在計算機科學領域中，排序演算法是資料處理與演算法分析的基礎。無論是處理巨量數據分析還是執行複雜的數值運算，演算法的效率直接影響系統的執行效能與即時反應。本報告旨在透過 JavaScript 實作五種經典排序演算法，並開發具備動態調速功能的視覺化界面，藉此直觀地觀察並比較各演算法在不同時間複雜度下的運行邏輯與表現差異。

## 二、 排序演算法介紹與實作連結

* **氣泡排序 (Bubble Sort)** 基本原理為重複走訪待排序序列，比較相鄰的兩個元素，若順序錯誤則進行交換。較大的數值會如同氣泡般逐一浮動至序列末端。  
    👉 [氣泡排序實作演示](./0309.html)

* **選擇排序 (Selection Sort)** 其操作方式是在未排序的數組中尋找最小值（或最大值），將其放置於已排序序列的末尾。此方法的主要特點是交換次數較少，但比較次數固定。  
    👉 [選擇排序實作演示](./selection.html)

* **插入排序 (Insertion Sort)** 透過建構有序序列，對於未排序數據，在已排序序列中從後向前掃描，找到相應位置並插入。在數據量小或接近已排序狀態時效率極高。  
    👉 [插入排序實作演示](./insertion.html)

* **合併排序 (Merge Sort)** 採用分而治之（Divide and Conquer）策略。將數組遞迴地拆分為最小單位，再將這些有序的子序列合併成一個完整的有序序列，具備穩定的運算效率。  
    👉 [合併排序實作演示](./merge.html)

* **沈澱排序 (Sedimentation Sort)** 此演算法為基於計數排序概念的自創變體。透過將數值直接映射至相對應的空間層級（地層），模擬顆粒依重量沈降的物理過程，避免了傳統演算法中的大量元素比較。  
    👉 [沈澱排序實作演示](./sedimentation.html)

## 三、 複雜度分析與效能比較

| 排序演算法 | 平均時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- |
| 氣泡排序 | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n^2)$ | $O(1)$ | 穩定 |
| 合併排序 | $O(n \log n)$ | $O(n)$ | 穩定 |
| 沈澱排序 | $O(n+k)$ | $O(k)$ | 穩定 |

## 四、 心得與結論
透過本次實作與數據模擬，可以觀察到不同複雜度等級演算法在效能上的巨大鴻溝。在數據規模較小時，氣泡與選擇排序雖易於實作且邏輯直觀，但當數據量提升至千位級別時，其運算時間會呈二次方增長，效率顯著下降。相較之下，合併排序展現了 $O(n \log n)$ 在處理大規模運算時的高度穩定性。在開發視覺化界面的過程中，透過調整「速度滑桿」，能以動態方式觀察演算法內部的運行過程，例如合併排序如何進行遞迴拆分與歸併。此外，自行設計的沈澱排序法也展現了在特定場景下，利用空間映射來優化排序效率的可能性。總結而言，選擇演算法時應根據資料規模、記憶體限制以及資料特性，在時間與空間複雜度之間做出最合理的權衡。
